OVW-CRSLinked-course cards — a horizontal scroller of .local-dimensions-course-card-lg. Each card: a 128px image (first valid get_course_overviewfiles() file) or, when absent, a gradient placeholder with getInitials(name) (first+last word initials, or first two chars if one word); a 3-line-clamped course name (fullname || shortname); an 8px progress track with a width: {progress}% fill; the {progress}% label; and a full-width Access button (core Boost btn btn-secondary) linking to /course/view.php?id={id}. Section title = linked_courses ("Linked courses") + a count badge (courses.length). Data source: the plugin WS local_dimensions_get_competency_courses runs its own SQL — SELECT DISTINCT c.id,… FROM {competency_coursecomp} cc JOIN {course} c ON c.id=cc.courseid WHERE cc.competencyid=:id AND c.visible=1 ORDER BY c.fullname — not tool_lp_list_courses_using_competency (its docblock still says "wraps" that, but execute() queries directly), then applies the enrolment-filter cascade (competency → plan template → global) via calculator::filter_courses_by_enrollment. accordion.js renderCourseCardsScrollable:1619; get_competency_courses.php:83-103; styles.css:1488-1672 · MIGRATES: placeholder gradient #667eea→#764ba2 (--lk-accent/--lk-accent-grad-end), progress fill #28a745 (--lk-success), badge text #5f6368 (--lk-pill-text), section platter #f1f3f4 (--lk-pill-platter) OVW-CRS-NAVScroll control group + 100% check. A bottom-right pill (role="group") of two arrow buttons — Prev (evidence_slider_prev = "Previous") and Next (evidence_slider_next = "Next"), each an aria-labeled icon button. Both are markup-rendered always but initScrollableTrack shows them only when shouldShowScrollableControls(itemCount) is true — itemCount > 2 (or mobile); ≤2 courses render .local-dimensions-courses-no-scroll (wrap, controls hidden). updateScrollableArrows toggles .disabled (opacity .45, #8da1b6, pointer-events none) when a track edge is reached — Prev starts disabled at scroll 0. Prev/Next scroll by one card (scrollToTrackAdjacentCard); the track is also drag-scrollable. The green fa-check-circle appears when progress >= 100; note its emitted class .local-dimensions-course-check has no CSS rule (the styled .local-dimensions-course-progress-check #198754 at styles.css:1250 is a different tracker class), so it inherits body colour. Focus ring on a nav button = outline:2px solid var(--dimension-custombgcolor, var(--primary, #667eea)) → --lk-accent (migrates). accordion.js renderCourseCardsScrollable:1664-1689, initCourseScroll:1702 → initScrollableTrack:1436 (shouldShowScrollableControls:1208, updateScrollableArrows:1399); styles.css:2439-2489
