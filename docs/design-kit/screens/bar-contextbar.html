Comportamento
Client-side, sem reloadPane (espelha o applyShowHidden da aba Estruturas): o servidor renderiza todas as opções — visíveis + ocultas-permitidas — marcando as ocultas com data-hidden="1"; por padrão o <select> mostra só as visíveis; ligar o toggle reconstrói a lista de opções a partir de um snapshot, preservando a seleção.
Edge: se a categoria selecionada persistida for oculta, o toggle inicia ligado — senão o contexto atual sumiria da lista.
Persistência · por-usuário (decisão)
O estado grava na preferência JSON central_nav (que já guarda contexto + categoria), como showhiddencats — sobrevive sessões e dispositivos, consistente com o resto do estado da barra. Toda chave nova na pref precisa entrar no sanitizador (helper::get_central_prefs), senão volta ao default. O provider de privacidade já cobre central_nav (sem string nova de privacidade).
Backend + a11y
central_category_options() passa a marcar cada opção com hidden (categoria visible=0) e a barra ganha um flag hashiddencategories para o gate de render. O <label for> é real (o named selector "checkbox" do Behat não casa por aria-label). Rótulo = string nova central_bar_showhiddencategories (en + pt_br).
Não é o "Mostrar estruturas ocultas". Aquele (FWK/EST) revela frameworks ocultos numa aba e persiste em sessionStorage; este revela categorias de curso ocultas no picker da barra e persiste na pref central_nav. Mesmo padrão visual, superfícies e dados diferentes.
