O corpo do detalhe são três métricas, não chips de contagem. structure_detail_content.mustache:75-111 desenha Cursos vinculados, Atividades vinculadas e Planos de aprendizagem vinculados — cada valor é um <button data-action="show-usage"> que abre o modal de uso (data-usage = courses / activities / templates), e vira <div> inerte quando linksclickable é falso. Daí o sublinhado no valor.
Os chips do cabeçalho são estes, todos hidden até terem valor (:51-70): regra — o único chip-accent, com fa-check —, rótulo (fa-tag), idnumber em monoespaçada, escala, Tag 1 e Tag 2. O texto sai com prefixo: central_structure_rule = "Regra: {$a}" e central_plans_labelchip = "Rótulo das competências: {$a}"; o prefixo do idnumber é a string idnumber do core, não do plugin. O chip "rótulo" é o campo personalizado Type (competency_detail.js:161-168), e a regra só aparece em nó com filhas e com ruletype — folha nunca mostra (:152-154). Não há chip de framework aqui: o chip colorido de framework é do paginated-picker, onde o local_dimensions_search_competencies devolve a tag de origem por resultado. Os valores acima são defaults reais — o tag1_options shipado é literalmente Edite-me 1/2/3.
O painel não tem botão de ação — e isso é de propósito. As sete ações da competência (Editar detalhes, Adicionar filha, Regra, Cursos e atividades, Competências referenciadas, Mover para, Excluir) moram no sticky-footer compartilhado da página (structure_footer_actions.mustache:41-68, um core\output\sticky_footer único montado em central.php:139-141). Ver sticky-footer.html.
O título do pane da esquerda é Competências: a aba de shortname structure é rotulada managecompetencies_structure = "Competências". "Estruturas" é a outra aba (central_frameworks_tab) — a nomenclatura interna do hub é invertida em relação aos rótulos visíveis.
